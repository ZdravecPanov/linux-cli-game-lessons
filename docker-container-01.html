Broken Container Startup Config

This lesson is a practical guide to container troubleshooting through state, logs, and runtime configuration. The goal is to repair the proven input, not to restart blindly.

Situation

An alert-router container keeps restarting. Your job is to inspect its runtime evidence, identify the bad env value, and prove the container becomes healthy after the fix.

Mission Goal

  • Inspect state and logs before editing config
  • Find the bad runtime env value
  • Fix the config and verify a healthy container

What Layer This Is

This mission lives in the container runtime layer: state, logs, restart behavior, and runtime env injection.

How To Use It in Practice

  • Start with container state and logs
  • Use inspect to verify what config reached runtime
  • Fix the env source, not random files
  • Restart and verify health

What You Should Learn

Container incidents still reward the same habit: prove the failing input before you restart.
