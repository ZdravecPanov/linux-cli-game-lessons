Linux CLI Game Lessons

Short field-guide style lesson pages for the Linux CLI Game. Each page explains the broken layer, investigation pattern, common mistakes, and what good operational behavior looks like.

sysadmin - intermediate

Missing ACL Access Grant

An analyst should have read access to one sensitive report without making the whole report world-readable. Standard mode bits are not the intended solution here. Use ACL-style evidence, grant the right access, and verify the analyst can read only through the intended narrow fix.

Lesson: acl-access-01.html

sysadmin - intermediate

Cron Job Fails on Schedule

An internal report-digest task is supposed to run from cron, but the scheduled run keeps failing while the script itself exists on disk. Prove why the job fails only in scheduled execution, repair the cron job safely, and verify the scheduled run becomes healthy.

Lesson: cron-failure-01.html

sysadmin - intermediate

Filesystem Running Out of Space

A storage alert says the operations volume is almost full and a cache maintenance check is failing. Prove where the space is going, remove only the safe obsolete data, and verify the volume health check passes again.

Lesson: disk-space-01.html

noc-l2 - intermediate

Internal Name Resolution Failure

An internal tool should answer at http://inventory.internal:8088/health, but users say the hostname does not work. You need to prove whether the failure is network reachability, local listening, or name resolution.

Lesson: dns-resolution-01.html

sysadmin - intermediate

Broken Container Startup Config

A small containerized alert-router workload keeps restarting. Use Docker-style evidence to find the bad runtime setting, repair the container config, and verify the workload becomes healthy again.

Lesson: docker-container-01.html

sysadmin - advanced

Final Exam - Multi-Layer Recovery

A small internal ops portal is failing in several layers at once. You need to restore service health, restore the internal hostname, and repair one narrow sudo delegation needed for support work. This mission is meant to verify that you can isolate layers instead of treating the whole incident as one mystery.

Lesson: final-exam-01.html

sysadmin - intermediate

Filesystem Full by Inodes

A ticket spool looks full even though it barely uses any disk space. The real problem is inode pressure caused by too many tiny obsolete files. Prove that the filesystem is full by inodes, clean the right path safely, and verify health returns.

Lesson: inode-pressure-01.html

sysadmin - intermediate

Find Failure in the Journal

A metrics forwarder service stopped after a cleanup job. The service manager confirms the unit is failing, but the actual reason is buried in the journal. Read the journal, find the startup blocker, repair it, and verify the unit is healthy again.

Lesson: journalctl-01.html

Log-Driven App Recovery

A lightweight internal reporting app keeps failing its startup check. You are told only that 'the app is broken again'. Use the app log to find the failure reason, fix the config, and prove the startup check is healthy.

Lesson: log-recovery-01.html

Lesson: permission-denied-01.html

A local API failed to come back after a test utility was left running on the same port. Prove that the problem is a listener conflict, identify the wrong process, free the port safely, and verify the real service is healthy again.

Lesson: port-conflict-01.html

A service is failing even though the executable path is correct. The real problem lives in the environment file the unit loads at startup. Find the bad value, repair the env file, and prove the service becomes healthy.

Lesson: service-envfile-01.html
