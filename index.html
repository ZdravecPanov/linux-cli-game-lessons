Linux CLI Game Lessons

Short field-guide style lesson pages for the Linux CLI Game. Each page explains the broken layer, investigation pattern, common mistakes, and what good operational behavior looks like.

sysadmin - intermediate

Filesystem Running Out of Space

A storage alert says the operations volume is almost full and a cache maintenance check is failing. Prove where the space is going, remove only the safe obsolete data, and verify the volume health check passes again.

Lesson: disk-space-01.html

noc-l2 - intermediate

Internal Name Resolution Failure

An internal tool should answer at http://inventory.internal:8088/health, but users say the hostname does not work. You need to prove whether the failure is network reachability, local listening, or name resolution.

Lesson: dns-resolution-01.html

sysadmin - intermediate

Find Failure in the Journal

A metrics forwarder service stopped after a cleanup job. The service manager confirms the unit is failing, but the actual reason is buried in the journal. Read the journal, find the startup blocker, repair it, and verify the unit is healthy again.

Lesson: journalctl-01.html

sysadmin - intermediate

Log-Driven App Recovery

A lightweight internal reporting app keeps failing its startup check. You are told only that 'the app is broken again'. Use the app log to find the failure reason, fix the config, and prove the startup check is healthy.

Lesson: log-recovery-01.html

sysadmin - intermediate

Broken Nginx Site Config

A small internal site stopped responding after a config change. The host should serve a simple health page on localhost:8080, but the web stack does not come up cleanly.

Lesson: nginx-config-01.html

sysadmin - refresh

Shared Directory Lockout

A teammate reports they cannot create a file in a shared operations folder. Your job is to prove whether the problem lives in file mode, directory traversal, ownership, or group access before making a fix.

Lesson: permission-denied-01.html

sysadmin - intermediate

Port Already in Use

A local API failed to come back after a test utility was left running on the same port. Prove that the problem is a listener conflict, identify the wrong process, free the port safely, and verify the real service is healthy again.

Lesson: port-conflict-01.html

sysadmin - intermediate

Service Fails to Start

A teammate says the report sync service never came back after a maintenance change. Use service status to identify why the unit is failing, fix the problem, and verify the service is healthy again.

Lesson: systemd-status-01.html
